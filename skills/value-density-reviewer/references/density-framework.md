# Density Framework

Use five complementary lenses. Diagnose them separately before deciding which tradeoff matters.

## 1. Visual density

How much appears in the available space. This is a fast perception, not a quality judgment.

Questions:

- What makes the interface feel full or sparse?
- Is that impression caused by content, decoration, spacing, contrast, or repetition?
- Does the visual impression match the amount of useful information?

## 2. Information density

How much of what is shown communicates useful information.

Questions:

- Which marks, labels, colors, and controls carry distinct information?
- Which elements repeat the same message without helping recognition or action?
- Is important information missing at the moment of decision?

## 3. Meaning density

How effectively arrangement communicates relationships and structure.

Inspect proximity, similarity, common region, hierarchy, sequence, and familiar patterns. Whitespace can carry meaning by separating groups.

Questions:

- Can the audience tell what belongs together?
- Does hierarchy reflect the task rather than decoration?
- Could rearrangement communicate more without adding content?

## 4. Time density

How quickly a person can reach, interpret, and act on useful information.

Inspect loading, navigation, repeated decisions, avoidable confirmations, feedback, recovery, and opportunities for safe batch action.

Questions:

- How many steps and decisions separate intent from outcome?
- Where does the interface make the user wait or repeat work?
- Does feedback match the actual duration and risk of an operation?

## 5. Value density

The useful outcome achieved relative to time, space, and attention requested. Treat this as an optimization lens, not a literal formula unless empirical measurements exist.

Questions:

- What becomes possible for this audience?
- Which density tradeoff increases the probability, quality, or speed of that outcome?
- What value would be lost by the proposed simplification or compression?

## Source Basis

The five-lens method is derived from Matthew Ström-Awn's “UI Density” (20 May 2024), available at https://mattstromawn.com/writing/ui-density/. The source discusses visual, information, design, temporal, and value density and attributes related work including Edward Tufte and Gestalt psychology.

This reference is a concise derived method with attribution. It does not reproduce the source article.
